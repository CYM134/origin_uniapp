## 项目概述

本方案对 uni-app 实验室预约管理系统进行全面 UI 规范。系统包含三个身份端（管理员、学生、教师）共 26 个页面，旨在解决布局松散、风格不统一、视觉割裂等问题。

**核心原则：仅修改样式文件（`.less` / `.css`），不触碰业务逻辑、接口逻辑、状态管理和功能交互。**

## 优化策略

> [!IMPORTANT]
> **渐进式优化**：仅修改 `.less` / `.css` 文件，保持 `.vue` 模板结构和 `<script>` 不变。  
> 通过全局变量（uni.scss）统一管理设计 Token，各页面样式引用变量而非硬编码颜色。

### 优化重点
1. **全局样式（uni.scss + app.less）**：引入设计变量体系
2. **导航栏（navigation-bar）**：三端颜色与样式统一
3. **各身份登录页**：现代感登录界面重新设计
4. **工作台首页**：卡片布局、欢迎区域升级
5. **列表类页面**：卡片样式、状态标签、空状态统一
6. **表单类页面**：输入框、选择器、按钮统一风格
7. **弹窗类**：圆角、阴影、按钮统一
8. **课表页面**：日视图/周视图容器优化

## 验证计划

### 每个页面修改后
- 确保 `.vue` 文件中的类名均有对应样式
- 确保无样式冲突或遮盖
- 验证移动端适配（375px/414px 常见宽度）

### 整体完成后
- 三端主题色一致性视觉检查
- 状态标签颜色统一性检查
- 弹窗样式一致性检查
- 空状态、加载状态样式检查
