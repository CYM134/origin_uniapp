# H5 发行指南

本工程是 **uni-app（Vue3）** 工程，一套代码本期发布到 H5 / 微信小程序（架构可拓展至 App）。
本文只讲 **H5 端如何"发行"成可部署的静态网站**，并交给 Nginx 托管。

> 名词区分：
> - **运行（run）**：调试用，产物在 `unpackage/dist/dev/`，带 sourcemap、体积大、未压缩。
> - **发行（build）**：正式用，产物在 `unpackage/dist/build/`，已压缩优化，才是要部署的东西。
>
> 目前仓库里只有 `unpackage/dist/dev/mp-weixin`（小程序调试产物），**H5 从未发行过**，
> 所以线上打开是 `deploy/html/index.html` 占位页。按下面任一方法发行后即可替换。

---

## 方法一：用 HBuilderX 发行（推荐）

本工程就是 HBuilderX 工程，这是最稳、最省事的方式。

### 1. 导入工程

HBuilderX → 文件 → 打开目录 → 选择前端工程目录：

```text
origin_uniapp/origin_uniapp
```

（注意是**内层**那个 `origin_uniapp`，里面有 `manifest.json`、`pages.json` 的才是工程根。）

### 2. 执行发行

顶部菜单 **发行 → 网站-PC Web 或手机 H5(仅适用于uni-app)**。

弹窗里：
- 网站标题：可填「实验室空间预约与协同管理系统」
- 网站域名：**本地部署留空即可**（留空 = 相对路径 `./`，配合 Nginx 放根目录能正常访问）
- 点「发行」

控制台出现 `Build complete.` 或打开产物目录即表示成功。

### 3. 找到产物

发行成功后产物在：

```text
origin_uniapp/origin_uniapp/unpackage/dist/build/web/
```

里面应有 `index.html`、`static/`、`assets/` 等。这就是完整的 H5 网站。

## 方法二：命令行发行（本工程暂不支持，说明如下）

> **本工程是 HBuilderX 工程，无法直接命令行发行 H5。** 已核实：
> `package.json` 的 `scripts` 为空、无 `vite.config.*`、无根 `index.html`，
> 不是 uni-app CLI 结构，`npm run build:h5` 会因找不到脚本而失败。
> **请用方法一（HBuilderX）。**

如果确实需要脱离 HBuilderX、用命令行/CI 构建 H5，需要先把工程**迁移为 uni-app CLI 工程**
（`npx degit dcloudio/uni-preset-vue#vite my-project` 生成 CLI 骨架，再把本工程的
`pages/`、`api/`、`static/`、`pages.json`、`manifest.json` 等迁入，补齐
`@dcloudio/vite-plugin-uni` 等依赖与 `build:h5` 脚本）。这属于工程改造，成本较高，
课程交付一般不必；需要的话我可以另外帮你做这套迁移。

## 部署：让 Nginx 托管发行产物

不论用哪种方法，拿到 `unpackage/dist/build/web/` 后：

### 1. 指向产物目录

修改仓库根目录 `.env`（没有就复制 `.env.example` 为 `.env`）：

```text
# 改这一行
NGINX_HTML_DIR=./deploy/html
# 改成（指向 H5 发行产物，注意两层 origin_uniapp）
NGINX_HTML_DIR=./origin_uniapp/origin_uniapp/unpackage/dist/build/web
```

### 2. 重启 Nginx

```bash
docker compose up -d --force-recreate nginx
```

### 3. 访问

```text
http://localhost
```

Nginx 已配置：
- 单页应用回退 `try_files $uri $uri/ /index.html`（刷新子路由不 404）
- `/api`、`/actuator`、`/swagger` 反向代理到后端

前端默认请求 `http://localhost:8080`（见 `origin_uniapp/api/request.js` 的 `API_BASE_URL`），
经 Nginx 同源反代通常无需改动。若换域名/端口，可运行时用
`uni.setStorageSync('apiBaseUrl', '你的后端地址')` 覆盖，无需重新发行。

## 常见问题

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| 页面空白 / 资源 404 | `NGINX_HTML_DIR` 少写一层 `origin_uniapp`，或产物目录为空 | 确认 `unpackage/dist/build/web/index.html` 存在，路径写全两层 |
| 接口 404 / 跨域 | 后端没启动，或没走 Nginx 反代 | 先 `docker compose up -d` 起后端，经 `http://localhost` 访问而非直接开 html 文件 |
| 改了代码页面没变 | H5 是发行时快照 | 重新发行，浏览器强制刷新（Ctrl/Cmd+Shift+R）清缓存 |
| 直接双击 index.html 打不开接口 | `file://` 协议无法反代 | 必须通过 Nginx（`http://localhost`）访问 |
