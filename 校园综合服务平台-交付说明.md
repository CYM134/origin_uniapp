# 校园综合服务平台 — 第一阶段交付说明

在原「实验室预约管理系统」基础上**增量扩展**为校园综合服务平台。严格遵循约束：复用现有登录/JWT/JdbcTemplate 体系与接口风格，未改动旧表、未重写登录、未删除实验室预约功能、未新增 super_admin（admin 在 P0 兼任超级管理员）。

> 本轮范围：P0 + P1 + P2 全部完成。

## 一、已完成能力

### P0 主链路
- **角色识别 + 菜单/按钮权限**：新增 `GET /api/auth/menus`、`GET /api/auth/permissions`，按 admin/teacher/student 返回菜单与按钮权限。

- **初始化数据**：9 应用分类 + 15 应用 + 2 演示公告 + 5 资讯分类 + 3 演示资讯 + 3 演示日程。

- **管理员工作台**：`GET /api/admin/portal/dashboard`（数字概览 + 任务统计 + 预约状态分布 + 通知/资讯中心 + 最近预约）；前端用纯 CSS 条形图（不引入 ECharts 重依赖，保证 uni-app 多端兼容）。

## 二、数据库脚本与执行顺序

脚本位于 `backend/docs/sql/`，**全部可重复执行**（`CREATE TABLE IF NOT EXISTS` / `INSERT IGNORE` / `WHERE NOT EXISTS`），**不修改/不删除任何旧表**。

| 顺序 | 文件 | 内容 |
|---|---|---|
| 1 | `V1__portal_app.sql` | `portal_app_category`、`portal_app`、`portal_user_app_favorite`、`portal_app_visit_log` |
| 2 | `V2__notice_news.sql` | `notice`、`notice_read`、`news_category`、`news` |
| 3 | `V3__calendar_ai.sql` | `calendar_event`、`ai_conversation`、`ai_message` |
| 4 | `V4__init_portal_data.sql` | 应用分类/应用/演示公告/资讯/日历初始化数据 |

> 说明：实验室预约（`reservation_applications`，44 列）已满足需求文档字段，**无需 V3 扩展旧表**，故未对旧表做任何 ALTER。

**执行方式**（本机无 mysql 客户端，可用任意 MySQL 客户端或项目自带 connector 执行）：
```bash
# 用任意 MySQL 客户端连接 application.yml 中配置的库，依次导入 V1→V2→V3→V4
mysql -h <host> -P <port> -u <user> -p <db> < backend/docs/sql/V1__portal_app.sql
# ... V2, V3, V4 同理
```
> 本轮已对 application.yml 配置的远程库执行 V1–V4 并验证通过（11 张新表、种子数据齐全、重复执行幂等）。

## 三、启动与测试

### 后端（Spring Boot 3.3 / JDK17）
```bash
cd backend
export JAVA_HOME=<JDK17路径>          # 本机：/opt/homebrew/Cellar/openjdk@17/17.0.17/...
```

## 六、已知限制与后续建议

1. **前端无法本机构建**：HBuilderX 工程，仅做了「Vue SFC 编译 + 脚本语法」静态校验，未在 HBuilderX 实跑；请在 HBuilderX 中验证交互与样式。
2. **图表**：工作台用 CSS 条形/进度条，未引入 ECharts（多端兼容优先）；后续如需更丰富可视化可接入 uCharts/ECharts。
3. **登录落地页**：现仍为原 `*-work` 页，门户作为其入口一键直达；如需「登录后直接进门户」，可在登录页重定向到 `pages/portal-home/portal-home`（建议 HBuilderX 验证门户无误后再切换）。
4. **权限模型**：P0 用 `visible_roles` 逗号串 + 静态菜单目录；如需 super_admin/细粒度 RBAC，再迁移到 `portal_app_role` 与角色/菜单表。
5. **AI**：默认走内置规则问答；接入真实模型只需配置 `AI_*` 环境变量，无需改码。
6. **JWT_SECRET**：生产务必固定，否则重启后旧 token 失效。
